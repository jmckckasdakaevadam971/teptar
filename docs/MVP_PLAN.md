# MVP-план — Тептар

> Цель документа: зафиксировать минимально жизнеспособную версию (MVP),
> которую можно показать пользователям и проверить спрос, не распыляясь на
> второстепенные фичи.

---

## 1. Видение продукта

Единая, достоверная и совместно поддерживаемая база родословных чеченских
тейпов. Главная ценность для пользователя — **найти своих предков и общих
родственников** с другими людьми, а для общества — сохранить родовую память
(традиция «знать семь предков» — *ворх дех*).

## 2. Целевая аудитория

| Сегмент | Потребность |
| --- | --- |
| Обычные пользователи | Посмотреть своё древо, найти общего предка с другим человеком |
| Старейшины / знатоки рода | Внести и выверить родословную тейпа |
| Администраторы тейпа (по сёлам) | Модерировать корректность данных |
| Исследователи / историки | Выгрузить данные (Excel/PDF/Visio) |

## 3. Роли пользователей

| Роль | Права |
| --- | --- |
| `viewer` | Просмотр approved-данных, поиск общих предков |
| `editor` | Предложить добавление/правку (уходит на модерацию) |
| `teip_admin` | Модерация персон своего тейпа в закреплённых сёлах |
| `super_admin` | Полный доступ, назначение админов, справочники тейпов/сёл |

## 4. Объём MVP (scope)

### ✅ Входит в MVP

1. **Карточка человека**: ФИО, пол, годы жизни, населённый пункт, примечание.
2. **Связи родитель→ребёнок** (отец обязателен для линии тейпа, мать опционально).
3. **Справочники**: тейпы, сёла (тукхум/гар — опционально, можно позже).
4. **Древо** — интерактивный просмотр предков и потомков выбранного человека.
5. **Поиск общих предков** между двумя выбранными людьми.
6. **Модерация**: статус `pending/approved/rejected`, очередь для `teip_admin`.
7. **Аутентификация** и роли.
8. **Экспорт**: Excel (таблица) и PDF (дерево). Visio — в виде совместимого
   формата (см. §8).

### ❌ НЕ входит в MVP (отложено)

- Полноценный нативный `.vsdx` для Visio (только совместимый экспорт).
- Загрузка фотографий и документов.
- Мобильное приложение (только адаптивный веб).
- Сложные брачные/усыновительные графы (только базовые браки).
- Система споров/голосований по конфликтующим версиям родства.
- Автоматическое выявление дубликатов (только ручное слияние админом).

## 5. Ключевые user stories

```text
US-1  Как пользователь, я хочу найти человека по ФИО/тейпу/селу,
      чтобы открыть его карточку.
US-2  Как пользователь, я хочу увидеть древо предков человека до N поколений.
US-3  Как пользователь, я хочу выбрать двух людей и узнать их общего предка
      и степень родства.
US-4  Как editor, я хочу добавить человека и связать его с отцом,
      чтобы дополнить родословную (запись уходит на модерацию).
US-5  Как teip_admin, я хочу видеть очередь правок по моему тейпу/селу
      и одобрять/отклонять их.
US-6  Как пользователь, я хочу выгрузить древо в Excel и PDF.
US-7  Как super_admin, я хочу назначить администратора тейпа на село.
```

## 6. Критерии готовности MVP (Definition of Done)

- [ ] Можно зарегистрироваться и войти; роли работают.
- [ ] Можно создать/отредактировать персону и связь с отцом.
- [ ] Правки `editor` попадают в очередь и видны `teip_admin`.
- [ ] Древо предков/потомков отображается и анимируется.
- [ ] Поиск общего предка возвращает корректный результат на демо-данных.
- [ ] Экспорт Excel и PDF работает.
- [ ] Демо-данные (несколько тейпов, 3–4 поколения) загружаются сидом.

## 7. Метрики успеха (после запуска MVP)

- Кол-во внесённых персон / неделя.
- Доля approved записей (качество модерации).
- Кол-во запросов «общий предок» (это вирусная фича-крючок).
- Кол-во активных администраторов тейпов.

## 8. Заметка про экспорт в Microsoft Visio

Нативный `.vsdx` — сложный OOXML-формат, готовых надёжных библиотек под Node
мало. Для MVP применяем один из совместимых путей:

1. **Excel-схема для Visio Data Visualizer** — выгружаем таблицу связей в
   спец-формате, Visio строит диаграмму автоматически. *(рекомендуется)*
2. **Legacy `.vdx` (XML)** — старый XML-формат Visio, который открывается
   современным Visio.

Полноценный `.vsdx` выносим в отдельную R&D-задачу пост-MVP
(см. [ROADMAP.md](ROADMAP.md)).

## 9. Риски MVP

| Риск | Митигация |
| --- | --- |
| Грязные/противоречивые данные | Модерация админами тейпов с самого начала |
| Дубликаты одного предка | Ручное слияние (super_admin), авто — позже |
| Приватность живых людей (152-ФЗ) | По умолчанию публичны только умершие; живые — ограниченная видимость и согласие |
| Низкая мотивация ввода | Фича «общий предок» как крючок + признание вклада |
| Чувствительность темы тейпов | Согласование правил с авторитетами, прозрачная модерация |
