# Дорожная карта — Тептар

Этапы развития продукта. Каждый этап — самостоятельная ценность.

---

## Этап 0 — Каркас (текущий)

- [x] Структура репозитория (backend / frontend / docs).
- [x] Документация: MVP-план, дизайн БД, архитектура.
- [x] DDL-схема PostgreSQL + демо-данные (seed).
- [x] Модульный скелет backend (persons, teips, villages, ancestors, export, auth).
- [x] Скелет frontend (древо, поиск, карточка).

## Этап 1 — MVP (ядро)

- [ ] CRUD персон + связи отец/мать.
- [ ] Справочники тейпов и сёл.
- [ ] Аутентификация + роли (RBAC).
- [ ] Модерация: очередь pending → approve/reject для `teip_admin`.
- [ ] Просмотр древа предков/потомков (анимация D3).
- [ ] Поиск общего предка (степень родства).
- [ ] Экспорт Excel + PDF.
- [ ] Назначение администраторов тейпа на сёла (`super_admin`).

**Критерий завершения:** см. Definition of Done в [MVP_PLAN.md](MVP_PLAN.md).

## Этап 2 — Качество данных

- [ ] Выявление и слияние дубликатов персон.
- [ ] Защита от циклов в связях (валидация на бэкенде).
- [ ] Журнал изменений с возможностью отката (`change_log`).
- [ ] Система споров: альтернативные версии родства + разрешение админом.
- [ ] Полнотекстовый поиск (pg_trgm) с опечатками.

## Этап 3 — Богатый контент и приватность

- [ ] Загрузка фото и сканов документов (object storage).
- [ ] Тонкая настройка приватности живых людей (152-ФЗ, согласия).
- [ ] Биография / значимые события в карточке.
- [ ] Привязка источников (кто подтвердил информацию).

## Этап 4 — Экспорт и визуализация (продвинутый)

- [ ] Нативный `.vsdx` для Microsoft Visio (R&D).
- [ ] Экспорт в GEDCOM (стандарт генеалогии — совместимость с другими ПО).
- [ ] Разные раскладки древа (веерная, вертикальная, радиальная).
- [ ] Печатные постеры древа (большой формат PDF).

## Этап 5 — Сообщество и рост

- [ ] Публичные страницы тейпов с статистикой.
- [ ] Уведомления: «к вашему роду добавлен человек».
- [ ] Геймификация вклада (признание активных знатоков рода).
- [ ] Мобильное приложение (переиспользовать стек DAKIR / Flutter).
- [ ] Интеграция входа через существующий Appwrite DAKIR.
